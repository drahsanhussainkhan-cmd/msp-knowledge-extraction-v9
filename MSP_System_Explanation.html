1. Problem Statement

Marine Spatial Planning (MSP) requires integrating knowledge from multiple sources: scientific research papers, national legislation, and environmental datasets. Currently, this integration is done manually by planners, which is:

  • Time-consuming: Reading hundreds of documents takes months
  • Error-prone: Important information is missed across document types
  • Language-fragmented: Turkish legal documents and English research papers are analyzed separately
  • Gap-blind: Disconnects between what research says, what law requires, and what data exists go unnoticed
Research Question: Can an automated NLP system extract structured knowledge from heterogeneous MSP documents (bilingual, multi-type) and detect cross-source gaps that would be invisible to manual review?

3. Extraction Pipeline

3.1 Extractor Design Pattern

Every extractor follows the same pattern:

  1. Inherit from BaseExtractor (abstract base class)
  2. Compile patterns: Define Turkish + English regex patterns in _compile_patterns()
  3. Extract: Iterate over pattern matches in document text
  4. Filter: Apply false positive filtering (marine relevance score, blacklists, context checks)
  5. Return: Typed dataclass instances (e.g., SpeciesExtraction, MethodExtraction)

3.3 Quality Control Measures

  • Marine Relevance Scoring: Every extraction is scored for marine context relevance (threshold: 0.05-0.10). Non-marine matches are rejected.
  • False Positive Filtering: Context-aware checks reject matches in bibliographies, author lists, CRediT statements, table headers.
  • Blacklists: Category-specific blacklists (e.g., species extractor rejects "Formal analysis", "Data curation" from CRediT author contributions).
  • Cross-line Rejection: Matches spanning newlines are rejected to prevent garbage captures.
  • Deduplication: Hash-based deduplication within each document prevents counting the same extraction twice.

4. Knowledge Base

All extractions are stored in a normalized SQLite database with three main tables:

DOCUMENTS (273 rows)
  - document_id, filename, doc_type, language, page_count

EXTRACTIONS (8,828 rows)
  - extraction_id, document_id, category, exact_text, context
  - page_number, confidence, marine_relevance, metadata (JSON)

CROSS_REFERENCES
  - Links species/methods/MPAs across document types
  - Enables integration gap detection

The CrossLinker module creates connections between:

  • Species mentioned in research papers AND in legal protection lists
  • Methods used in research AND referenced in regulations
  • MPAs designated in law AND studied in research

5. Gap Detection (Novel Contribution)

This is the core novelty of the system. While individual extraction is useful, the real value is detecting what's MISSING by cross-referencing across document types. No existing MSP tool does this automatically.

5.1 Four Gap Detection Dimensions
